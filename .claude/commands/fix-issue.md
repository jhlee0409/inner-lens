# GitHub 이슈 수정

GitHub 이슈를 분석하고 수정해주세요: $ARGUMENTS

## 단계

1. **이슈 분석**
   - 이슈 내용 파악
   - 재현 조건 확인
   - 영향 범위 파악

2. **코드 탐색**
   - 관련 파일 찾기
   - 근본 원인 분석
   - 기존 테스트 확인

3. **수정 구현**
   - 최소한의 변경으로 해결
   - 기존 패턴 따르기
   - 사이드 이펙트 방지

4. **테스트**
   - 회귀 테스트 작성
   - `npm run typecheck` 실행
   - `npm run test` 실행

5. **완료**
   - 변경 사항 요약
   - 커밋 메시지 제안 (fix: 형식)

## 규칙
- 요청하지 않은 리팩토링 금지
- 관련 없는 파일 수정 금지
- TODO 주석 남기지 않고 완전히 구현
